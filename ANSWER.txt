## 🎯 The Complete API

### Endpoint 1: Send Invite
```
POST /api/v1/invites
→ Recruiter sends invite with time slots
→ Creates notification for candidate
→ Returns invite_id
```

### Endpoint 2: Get Invite Details
```
GET /api/v1/invites/{invite_id}
→ Returns invite with proposed_time_slots
→ Used by frontend to show available slots
```

### Endpoint 3: Candidate Responds
```
POST /api/v1/invites/{invite_id}/respond
→ Candidate selects time slot
→ Updates database
→ Creates recruiter notification
```

### Endpoint 4: Get Notifications
```
GET /api/v1/notifications
→ Returns all notifications for user
→ Includes invite_id in metadata! (KEY FOR UI)
```

### Endpoint 5: Mark Notification Read
```
POST /api/v1/notifications/{id}/read
→ Mark notification as read
```

### Endpoint 6: Get Recruiter Responses
```
GET /api/v1/recruiter/responses
→ Recruiter views all candidate responses
→ Shows name, email, job title, selected slot, status
```
